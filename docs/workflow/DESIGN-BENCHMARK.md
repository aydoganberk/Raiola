# DESIGN BENCHMARK

- Workflow root: `docs/workflow`
- Product category: `Analytics Platform`
- Page type: `Dashboard`
- Reference blend: `Linear Precision structure + VoltAgent Command Energy type cues + VoltAgent Command Energy accent restraint`

## Benchmark Axes

### Hierarchy and shell

- Benchmark lead: `Linear Precision`
- Target: Use Summary rail plus Filters and scoped controls as the visible hierarchy spine.
- Review prompt: Can a new visitor or operator tell what the page is for and what to do next within one scan?

### Product proof

- Benchmark lead: `Linear Precision`
- Target: Primary metrics should stay anchored, not move around between breakpoints.
- Review prompt: Does real product proof arrive before decorative storytelling?

### Typography and copy

- Benchmark lead: `VoltAgent Command Energy`
- Target: Operator Dense copy: concise, directive, confident, and low-noise.
- Review prompt: Would the page still feel specific and authored if color were removed?

### State completeness

- Benchmark lead: `Analytics Platform`
- Target: Cover loading, empty, filtered-empty, error, success, destructive-confirmation as first-pass states, not backlog polish.
- Review prompt: Do critical states feel designed, or only technically handled?

### Component discipline

- Benchmark lead: `Table`
- Target: Favor the active custom stack before introducing a second component vocabulary.
- Review prompt: Are we reusing a coherent component vocabulary or inventing per-section variants?

### Primary outcome

- Benchmark lead: `Dashboard`
- Target: Support fast scanning, decisions, and safe next actions.
- Review prompt: Does the page keep momentum toward the primary outcome without hiding trust cues or next actions?

## Differentiation Plays

### Anchored hero moment

- Move: Give each primary screen one unmistakable anchor: a hero metric rail, authored hero block, or command surface.
- Why: Use one intentional signature moment so the page has a memorable point of view without becoming noisy.

### Lead with product proof

- Move: Primary metrics should stay anchored, not move around between breakpoints.
- Why: External-site frontend work feels cutting-edge when the product is concrete, not when the hero is more abstract.

### Borrow from Linear Precision

- Move: Use crisp containment and obvious hierarchy instead of oversized chrome.
- Why: This is the strongest reusable benchmark cue from the selected reference blend.

### Differentiate through structure

- Move: Ship build summary rail as a shared system primitive, not a page-local one-off.
- Why: Reusable structure compounds across new site pages faster than isolated visual polish.

### Constrain motion

- Move: Reserve motion for state change, selection, and live status.
- Why: The product will feel more premium when motion supports sequencing instead of shouting for attention.

## Commodity Risks

- Do not replace relational data with decorative cards when comparison matters.
- Do not bury risk and action behind accordions or hidden secondary tabs.
- Do not replace data relationships with oversized decorative cards.
- Do not bury status, risk, or primary actions below fold-heavy hero chrome.
- Do not mix multiple visual metaphors on one screen.
- Do not hide core actions in tertiary menus when the task is frequent.
