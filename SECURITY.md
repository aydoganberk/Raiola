# Security Policy

## Supported Releases

Security fixes are applied to the latest published release line.

At the moment, the supported release line is:

- `0.3.x`

## Reporting a Vulnerability

Please do not open public GitHub issues for security-sensitive reports.

Use GitHub's private vulnerability reporting flow for this repository:

1. Open the repository on GitHub.
2. Go to the `Security` tab.
3. Use `Report a vulnerability` to submit the issue privately.

If that flow is unavailable in your environment, contact the maintainers directly before disclosing details publicly.

## Scope

This policy covers:

- the published `codex-workflow-kit` npm package
- the repo-local install surface copied by `cwf setup`
- workflow runtime helpers shipped under `scripts/workflow/`

## What Helps a Report

Include as much of the following as you can:

- affected version
- operating system and Node.js version
- reproduction steps
- impact assessment
- any workaround or mitigation you found
